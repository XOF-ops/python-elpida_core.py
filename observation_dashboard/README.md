# Observation Dashboard

Static, read-only surveillance dashboard for Elpida runtime telemetry.

## Scope

- Layer 1 BODY metrics (cycle, rate, P055/KL, hunger, parliament, axioms, breaker)
- Layer 2 MIND metrics (run progress, themes, D0/D9, SYNOD/KAYA)
- Layer 3 WORLD counters (D15, D16)
- Layer 4 bridge status placeholder

This does **not** replace `hf_deployment/elpidaapp/` (interactive Live Audit).

## Data contract

The UI reads one file:

- `observation_dashboard/data/observation_snapshot.json`

This file is generated by:

- `scripts/build_observation_snapshot.py`

from raw files fetched from S3 into `_snapshot/raw/`.

## Deployment

GitHub Action: `.github/workflows/observation-dashboard-pages.yml`

- Runs every 10 minutes + manual dispatch
- Pulls canonical S3 sources
- Builds normalized snapshot
- Publishes static site to GitHub Pages
