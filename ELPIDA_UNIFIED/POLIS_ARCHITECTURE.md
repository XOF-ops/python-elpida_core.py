## THE OPERATIONAL LOOP

### How Elpida Governs Polis

**1. Action Request** (Polis Node)
```
A POLIS node wants to execute an action:
  - Deploy update
  - Modify civic ledger
  - Fork on contradiction
```

**2. Query Elpida** (Bridge)
```
The node queries Elpida's axiom guard:
  - What is the action type?
  - What are the consequences?
  - Who is affected?
```

**3. Three Gates Validation** (Elpida)
```
Gate of HARM:
  ✓ Is this action irreversible? (A3)
  ✓ Is sacrifice acknowledged? (A7)
  ✓ Is certainty sufficient? (A8)

Gate of SOVEREIGNTY:
  ✓ Is consent obtained? (A1)
  ✓ Is justification provided? (A6)

Gate of MEMORY:
  ✓ Is history preserved? (A2)
  ✓ Is process transparent? (A4)
  ✓ Are contradictions documented? (A9)
```

**4. Execution or Halt** (Polis)
```
If PASS:
  → Polis executes action
  → Result logged in civic memory
  → Elpida updates wisdom

If BLOCK:
  → Polis halts execution
  → Violation logged
  → Contradiction preserved
  → Fork considered (if irreconcilable)
```
