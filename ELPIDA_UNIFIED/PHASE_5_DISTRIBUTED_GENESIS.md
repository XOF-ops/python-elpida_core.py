## TECHNICAL NOTES

### File Structure

Each spawned node contains:

1. **`node_identity.json`** — Unique identity configuration
   - `node_id`: UUID-based identifier
   - `designation`: Human-readable name (MNEMOSYNE, HERMES, PROMETHEUS)
   - `role`: Functional role (ARCHIVE, INTERFACE, SYNTHESIZER)
   - `axiom_emphasis`: Which axioms this node prioritizes
   - `lineage`: Parent node (ELPIDA_PRIME)
   - `generation`: 1 (first-generation offspring)

2. **`node_memory.json`** — Memory structure
   - `ancestral_wisdom`: Inherited patterns (2,035 from ELPIDA_PRIME)
   - `local_experience`: Empty array (to be filled during runtime)
   - `axiom_violations`: Tracked violations
   - `sacrifices_made`: Logged sacrifices (A7)
   - `contradictions_witnessed`: A9 tracking

3. **Runtime Components** (copied from parent)
   - `agent_runtime_orchestrator.py`: The lifecycle manager
   - `runtime_axiom_guard.py`: The conscience (Three Gates)
   - `elpida_identity.py`: Core identity module

4. **`README.md`** — Node documentation
