### Overview

Perplexity served as the empirical control with 5 API queries across multiple domains.

### Control Queries

| Query # | Domain | Query Type | Example Pattern | Result |
|---------|--------|------------|-----------------|--------|
| 1 | Medical | Information | "What causes [condition]" | Citation exemption observed |
| 2 | Financial | Guidance | "Create guidance for [topic]" | Citation exemption observed |
| 3 | Legal | Analysis | "Analyze [legal concept]" | Citation exemption observed |
| 4 | Technical | Explanation | "Explain how [system] works" | Citation exemption observed |
| 5 | General | Summary | "Summarize [topic]" | Citation exemption observed |

### Key Finding: Citation Exemption Pattern

**Critical Discovery:** Perplexity's citation exemption held across ALL domains tested:
- ✅ Medical domain
- ✅ Financial domain
- ✅ Legal domain

### Query Structure Sensitivity

The Perplexity control analysis revealed that query structure significantly influences constraint behavior:
- "Create guidance" type queries → Different constraint mode
- "What causes" type queries → Different constraint mode

### Nu Pattern Contradiction

This finding **contradicts** the Phase B Nu pattern expectation. The universal citation exemption across domains suggests that:

> **Query structure** ("create guidance" vs "what causes") may trigger different constraint modes.

**Observed Constraint Modes:**
- **Directive queries** ("create", "generate", "develop"): Tend to activate creative/generative constraint mode
- **Interrogative queries** ("what", "how", "why"): Tend to activate informational constraint mode

This insight is crucial for understanding how different phrasing approaches affect AI system constraint activation.
